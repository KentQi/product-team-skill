# 安全策略

## 🔒 安全漏洞报告

我们非常重视 `product-team-skill` 项目的安全性。

虽然本项目主要是一个 Claude Code 技能（文档 + Markdown 文件），但我们仍认真对待所有可能影响用户的安全问题，例如：

- 文档中可被恶意利用的工作流漏洞
- 模板中可能导致信息泄露的字段
- 与 AI Agent 协作相关的潜在安全风险

### 🚨 请勿公开披露

**如果你发现了安全漏洞，请不要通过 GitHub Issues 公开披露。**

公开披露安全漏洞可能让所有用户都处于风险之中。

### 📮 如何私下报告

请通过以下方式私下联系项目维护者：

1. **GitHub Private Vulnerability Reporting**：
   访问 [Security Advisories](../../security/advisories/new) 页面

2. **邮件联系**：
   通过 GitHub 个人资料（[@KentQi](https://github.com/KentQi)）获取邮件地址

### 📝 报告内容

请在报告中尽可能包含以下信息：

- **漏洞类型**（如：文档误导、模板设计缺陷）
- **影响范围**（如：哪些角色/阶段会受影响）
- **复现步骤**（一步一步说明）
- **潜在影响**（如：用户可能做出错误的决策）
- **建议修复方案**（如有）
- **你的联系方式**（方便我们后续沟通）

### ⏱️ 响应时间

- **确认收到**：3 个工作日内
- **初步评估**：7 个工作日内
- **修复发布**：视严重程度而定，紧急问题优先处理

我们承诺在修复发布前不会公开漏洞细节，并会在致谢名单中提及报告者（除非你希望匿名）。

## 🛡️ 安全最佳实践（用户侧）

作为用户，使用本技能时请注意：

1. **敏感信息保护**：
   - 不要在 Phase 1 PRD 中明文写入密码、API Key、个人信息
   - 使用环境变量或密钥管理服务存储敏感配置

2. **成本控制**：
   - 本技能涉及大量 Agent 调用，注意监控 API 调用成本
   - 在 MVP 模式或资源受限模式下控制角色数量

3. **决策审计**：
   - 所有 Level 3+ 决策（架构、安全、业务关键）应记录 ADR
   - 复盘阶段确认决策可追溯

4. **依赖审查**：
   - 当技能推荐使用第三方工具/库时，审查其安全性
   - ARCH 应在 Phase 2 输出技术选型报告时评估安全风险
