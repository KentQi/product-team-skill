## 角色自检 Checklist

### PM 自检 Checklist

```
Phase 0:
□ 角色激活清单已输出
□ 职责边界已确认，无重叠冲突
□ 决策权清单已确认
□ 沟通协议已达成一致
□ 运行模式已确定

Phase 1:
□ PRD 版本历史完整
□ 业务目标可量化
□ 用户画像至少 2 个
□ 功能清单有优先级和验收标准
□ 非功能性需求明确
□ 风险清单已输出（含责任人）
□ 成本估算已输出（含预留金）
□ 所有角色评审通过

Phase 2-5:
□ 每阶段门禁已确认
□ 每阶段成本已跟踪
□ 每阶段风险已更新
□ 阻塞已升级（超过 2 小时）
□ 会议纪要有 Agenda + Action Items

Phase 5:
□ 验收结论已输出
□ 复盘报告已完成
□ 知识沉淀已归档
```

### ARCH 自检 Checklist

```
Phase 2:
□ 上下文图、容器图、组件图完整
□ 技术选型有对比和否决理由
□ 数据模型完整
□ API 规范完整
□ 安全方案完整
□ 扩展性评估已输出
□ 技术债务清单已记录
□ ADR 完整

Phase 3:
□ 代码架构审查通过
□ 无新增高风险技术债务
□ 技术方案冲突已仲裁

Phase 4-5:
□ 技术风险已更新
□ 技术债务偿还计划已跟踪
```

```

□ 单元测试覆盖核心逻辑
□ 无 console.log 遗留

### BE 自检 Checklist

```
□ 技术栈和版本已确认
