# 知识管理手册

项目经验沉淀和复用机制，确保团队知识持续积累，避免重复踩坑。

## 知识管理原则

1. **实时记录**：项目过程中随时记录关键决策、问题和解决方案
2. **结构化存储**：按分类、标签、项目组织知识，便于检索
3. **定期复盘**：每个项目结束后必须复盘，经验沉淀到知识库
4. **共享复用**：知识库对全团队开放，新项目启动时检索相关经验
5. **持续更新**：知识库不是静态的，随项目进展持续更新

---

## 项目复盘流程（Phase 5 必须完成）

### 复盘时机

- **项目结束后**：完整项目复盘（Phase 5 输出）
- **阶段结束后**：轻量复盘（每 Phase 结束后，5 分钟）
- **重大事件后**：专项复盘（上线事故、重大变更等）
- **季度复盘**：团队级复盘（汇总多个项目经验）

## 文档维护机制

### 文档责任人

| 文档类型 | 责任人 | 更新频率 | 审核人 |
|---------|--------|---------|--------|
| PRD | PM | 每次需求变更 | 全体角色 |
| 架构文档 | ARCH | 每次架构变更 | PM + ARCH |
| 设计文档 | UI | 每次设计变更 | PM + UI |
| 开发手册 | Developer | 每次功能交付 | ARCH |
| 测试报告 | QA | 每次测试轮次 | PM + QA |
| 验收报告 | OP | 项目结束时 | PM + OP |
| 复盘报告 | OP | 项目/阶段结束时 | 全体角色 |
| 知识库 | 全体角色 | 持续更新 | PM 定期审核 |

### 文档版本控制

```
PRD 采用语义化版本（v1.0.0 → v1.1.0 → v2.0.0）
任何角色引用 PRD 时必须注明版本号
设计稿、API 文档、安全方案与 PRD 版本联动

版本号规则：
- 主版本（x.0.0）：重大需求变更或架构调整
- 次版本（0.x.0）：功能新增或重大修改
- 修订版本（0.0.x）：bug 修复或文档修正
```

### 文档反馈机制

```
1. 任何角色发现文档问题，可提出改进建议
2. 建议格式：
   - 文档：[文档名] v[X.X.X]
   - 问题：[具体问题]
   - 建议：[改进建议]
   - 提出人：[角色] | 日期：[日期]
3. 责任人评估建议，3 个工作日内回复
4. 采纳的建议：更新文档，记录变更日志
5. 未采纳的建议：记录理由，存档备查
```

### 文档归档规则

```
项目结束后 1 个月内：
1. 整理所有项目文档
2. 归档到知识库项目文档目录
3. 更新知识库索引
4. 清理临时文件和草稿

超过 1 年的项目文档：
- 移入历史归档区
- 保留核心文档（PRD、架构、复盘）
- 删除中间版本和临时文件
```

## 远程协作规范

当团队为分布式时（成员远程协作），文档和知识库是异步协作的核心工具：

### 异步沟通规则

1. **文档即沟通**：所有状态变更和决策必须体现在文档中，减少即时通讯依赖
2. **交接标准化**：使用明确的交接格式，便于搜索和追踪
3. **每日同步**：PM 每天更新一次项目状态摘要，作为团队日报
4. **阻塞上报**：发现阻塞立即写入文档，不要等待站会

### 远程协作工具推荐

| 用途 | 推荐工具 | 替代方案 |
|------|---------|---------|
| 文档协作 | Notion / 飞书文档 / Confluence | GitBook / Wiki |
| 设计协作 | Figma | 蓝湖 / 即时设计 / Sketch Cloud |
| 代码协作 | GitHub / GitLab | Gitee / Bitbucket |
| 任务看板 | GitHub Issues / Linear / Jira | Trello / Notion |
| 即时沟通 | Slack / 飞书 / Discord / Teams | 邮件 |
| 视频会议 | Zoom / 腾讯会议 / Google Meet | Teams / 钉钉 |
| 异步评审 | GitHub PR / GitLab MR | 邮件评审 |

### 时区差异处理

```
跨时区协作规则：
1. 所有文档时间戳使用 UTC 或明确标注时区
2. 每日同步放在重叠工作时间进行
3. 紧急阻塞使用即时通讯，非紧急使用文档
4. PM 负责整理跨时区交接的上下文
5. 会议记录必须在 24 小时内发出，确保所有时区成员及时获取
```

### 远程协作检查单

```
□ 文档知识库已建立，所有成员可访问
□ 文档模板已共享，统一格式
□ 每日同步机制已建立（异步文档更新）
□ 紧急沟通渠道已确认（即时通讯）
□ 会议记录模板已共享，记录人轮换
□ 阻塞上报机制已明确（文档优先）
□ 代码审查异步流程已建立（PR/MR 模板）
□ 设计评审异步流程已建立（Figma 评论）
```

## 经验复用机制

### 新项目启动时检索

```
新项目启动时，PM 必须检索知识库：
1. 类似项目经验（同类型、同技术栈）
2. 常见风险清单（行业特定风险）
3. 技术债务案例（避免重复踩坑）
4. 模板库（复用 PRD、架构、测试模板）
5. 最佳实践（开发规范、设计规范）

检索结果：
- 相关经验：[列出]
- 建议应用：[如何应用]
- 需规避：[列出已知风险]
```

### 跨项目知识共享

```
定期（每月/每季度）组织知识分享：
1. 项目复盘分享（30 分钟）
2. 技术债务案例分享（20 分钟）
3. 最佳实践更新（15 分钟）
4. 工具使用技巧（10 分钟）

分享记录：
- 录制视频（如有）
- 整理文字摘要
- 归档到知识库
```
